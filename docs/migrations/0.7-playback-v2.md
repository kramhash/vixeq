# Migrating to Playback v2 (`0.7.0`)

- Status: Planned breaking migration
- First prerelease: `0.7.0-beta.1`
- Contract: [`../behavior/playback-v2.md`](../behavior/playback-v2.md)

Playback v2 removes clock-domain ambiguity and standardizes controls across
Core, React, and Player React. There are no deprecated aliases in 0.7; migrate
before upgrading.

## Package-wide renames and removals

| 0.6 API | 0.7 API/action |
| --- | --- |
| `SequencerTransport` | `PlaybackTransport` |
| `SequencerClock` | `PlaybackClock` |
| `TransportEvent` | `EnginePlaybackEvent` |
| `ArrangementTransportEvent` | `EnginePlaybackEvent` |
| `AudioClock` / `AudioContextClock` | Remove; use `PlaybackTransport` |
| `AudioClockOptions` | Transport-specific options |
| `createAudioClock()` | `createMediaElementTransport()` or a transport factory |
| `createAudioContextClock()` | `createAudioBufferTransport()` or `createClockTransport()` |
| Engine `clock` option | Engine `transport` option |
| `timeDriven` option | Remove; playback is always time-driven |
| `originMs` option | Remove; Engine owns logical anchoring |
| `stopAtMs` transport option | Remove; `stop()` always returns to 0 |
| `event.timestamp` | Scheduled/current transport-position fields |
| `ProjectEvent.timestamp` | `positionMs` and logical beat/step fields |
| `ArrangementSectionEvent.timestamp` | Scheduled/current position fields and `lateByMs` |

## Engine controls

| 0.6 | 0.7 |
| --- | --- |
| `engine.start()` | `await engine.play()` |
| `engine.stop()` used as pause | `await engine.pause()` |
| `sequencer.reset()` while preserving play/pause state | `await sequencer.seekStep(0)` |
| `sequencer.reset(stepIndex)` | `await sequencer.seekStep(stepIndex)` |
| old stop-then-reset sequence | `await engine.stop()` |
| `arrangement.seek(beat)` | `await arrangement.seekBeat(beat)` |
| `arrangement.reset()` while preserving play/pause state | `await arrangement.seekBeat(0)` |
| direct media scrub | `await engine.seekPositionMs(positionMs)` |
| `isPlaying()` | `getPlaybackState() === "playing"` |
| `engine.setBpm(bpm)` | `engine.setProject(setProjectBpm(engine.getProject(), bpm))` |
| Engine `"transport"` event | Engine `"playback"` event |
| `onTransportChange` | `onPlaybackChange` |

Before:

```ts
const engine = new SequencerEngine(project, {
  clock: audioClock,
  timeDriven: true,
  originMs: 0,
});

engine.start();
engine.stop();
engine.reset(4);
```

After:

```ts
const transport = createMediaElementTransport(audio);
const engine = new SequencerEngine(project, { transport });

await engine.play();
await engine.pause();
await engine.seekStep(4);
await engine.stop();
```

Engine controls are asynchronous because media operations can fail. Invalid
arguments still throw synchronously; valid operations may reject.

Old `reset()` preserved whether playback was running. Use unit seek to retain
that behavior. Use the new `stop()` only when the intended result is both
halted playback and position 0.

Arrangement controls follow the same async model:

```ts
const engine = new ArrangementEngine(arrangement, { transport });

await engine.play();
await engine.pause();
await engine.seekBeat(16);
await engine.stop();
```

`ArrangementEngine` local looping is configured with `loop` or
`setLoop(boolean)`. It does not mutate `transport.setLoop()`, and a local
Arrangement end does not stop a shared transport.

## Sampling

Before:

```ts
const values = engine.sampleChannels(performance.now(), easeOutQuad);
```

After:

```ts
const current = engine.sampleChannels(easeOutQuad);
const preview = engine.sampleChannelsAt(2_000, easeOutQuad);
```

`sampleChannels()` reads the Engine's transport position and works with
browser, media, and AudioBuffer transports. `sampleChannelsAt()` accepts
Project-relative elapsed milliseconds, not a rAF or AudioContext timestamp.

## Step and playback events

Remove consumer-maintained timestamp interpolation:

```ts
// 0.6 — clock-domain-sensitive
const phase = (performance.now() - event.timestamp) / event.durationMs;
```

and sample through the Engine:

```ts
// 0.7
const values = engine.sampleChannels(easeOutQuad);
```

When handling events, use `scheduledPositionMs` as the intended step boundary,
`transportPositionMs` as the dispatch-time position, `lateByMs` for
diagnostics, and `cause` to distinguish natural ticks, play, seek, and
Project-driven repositioning.

Arrangement section listeners now receive logical position fields instead of a
clock timestamp:

```ts
engine.on("section", (event) => {
  console.log(event.section?.id, event.beat, event.positionMs, event.cause);
});
```

## React hooks

| 0.6 | 0.7 |
| --- | --- |
| `isPlaying` | `playbackState` |
| `isStarting` | `pendingOperation` and `isBusy` |
| one `error` in `useArrangement` | `projectError` and `transportError` |
| `currentStep` as an independent source | derive from Engine position/latest event |
| `play`, `stop`, `toggle`, `reset` | `play`, `pause`, `stop`, `toggle`, unit seek |
| `clock`/`originMs` hook options | `transport` |
| `onTransportChange` hook/Player prop | `onPlaybackChange` |

Before:

```tsx
const player = useSequencerEngine({
  project,
  transport,
  timeDriven: true,
});

await player.stop(); // previously pause-like
await player.reset(0);
```

After:

```tsx
const player = useSequencerEngine({ project, transport });

await player.pause();
await player.stop(); // returns to position 0
```

Continuous progress uses `positionRef.current` or `onPosition`; it is not a
per-frame React state update.

## Player React

`SequencePlayerRef` removes `reset()` and adds:

- `play()`
- `pause()`
- `stop()`
- `toggle()`
- `seekStep(stepIndex)`
- `seekPositionMs(positionMs)`
- `setPlaybackRate(rate)`
- `setTransportLoop(loop)`

`SequencePlayerTransportState` replaces `currentStep`, `isPlaying`, and
`isStarting` with `playbackState`, `positionRef`, `pendingOperation`, and
`isBusy`. It keeps `latestEvent` and `transportError`, and adds
`projectError`.

Built-in transport controls become a Play/Pause toggle and a separate Stop
button. Stop returns to position 0.

Player props migrate as follows:

```tsx
// 0.6
<SequencePlayer
  project={project}
  onProjectChange={onProjectChange}
  clock={clock}
  timeDriven
  originMs={0}
  onTransportChange={onTransportChange}
/>

// 0.7
<SequencePlayer
  project={project}
  onProjectChange={onProjectChange}
  transport={transport}
  onPlaybackChange={onPlaybackChange}
/>
```

`StandaloneSequencePlayerProps` uses the same transport and callback changes.

## Envelopes and reduced motion

Add `reset()` to custom Envelope implementations:

```ts
const envelope: Envelope = {
  trigger(positionMs, value) {},
  sample(positionMs) { return 0; },
  reset() {},
};
```

Envelope timestamps are logical transport positions. Do not pass
`performance.now()`.

Replace:

```ts
useAnimatedChannels(engine, { reducedMotion: true });
```

with:

```ts
useAnimatedChannels(engine, { motionPreference: "reduce" });
```

The default is `"system"`; use `"no-preference"` to opt out of OS reduced
motion. In reduced mode, `useAnimatedChannels()` stops rAF and ignores
ordinary step ticks, but explicit seek, stop, and Project changes still
produce one fresh static sample.

`useAnimatedChannels()` no longer accepts `latestEvent` without an Engine.
Envelope mode now requires a `ChannelSource` so triggers, sampling, seek/stop
resets, and Project-change resets all use logical transport positions.

## Validation changes

`SequencerEngine`, `ArrangementEngine`, and Project update methods no longer
normalize malformed typed input. They validate and throw `TypeError`.

Normalize untrusted JSON explicitly before Engine construction:

```ts
const project = normalizeProject(JSON.parse(serialized));
const engine = new SequencerEngine(project, { transport });
```

Seek and rate inputs are no longer clamped or modulo-normalized. Clamp user
input in the UI, then call the strict Core method.

## Ownership and cleanup

An Engine borrows a supplied transport:

```ts
const transport = createMediaElementTransport(audio);
const engine = new SequencerEngine(project, { transport });

engine.dispose();
// transport is still valid
transport.dispose(); // caller owns this cleanup
```

This rule is required for shared Engines and React StrictMode remounts.
