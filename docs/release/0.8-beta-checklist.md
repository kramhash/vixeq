# 0.8 Beta Release Checklist

T7 prepares the v1-to-v2 migration fixtures and beta smoke gate for
`0.8.0-beta.1`. It does not publish the beta; publishing happens only after this
checklist passes.

## Preconditions

- T0-T7 are `done` in `docs/plans/v1-collaboration-spec.md`.
- Public package versions are bumped in lockstep to the chosen prerelease
  version, for example `0.8.0-beta.1`.
- `CHANGELOG.md`, package READMEs, API docs, and
  [`../migrations/0.8-timeline-arrangement-v2.md`](../migrations/0.8-timeline-arrangement-v2.md)
  describe the Timing/Timeline/Arrangement v2 migration behavior.
- The working tree contains no unrelated release artifacts such as `.tgz`,
  `.tmp/`, or copied example output.

## Required Local Gates

Run these from the repository root before publishing a beta:

```sh
pnpm -r --no-bail typecheck
pnpm -r --no-bail test -- run
pnpm -r --no-bail build
pnpm smoke:pack
```

`pnpm smoke:pack` builds `@vixeq/core`, `@vixeq/react`, and
`@vixeq/player-react`, packs them with `pnpm pack`, installs those tarballs into
a temporary workspace under `.tmp/pack-smoke`, and verifies:

- `@vixeq/core` ESM import from the tarball.
- `@vixeq/core` CJS require from the tarball.
- `@vixeq/core` v1-to-v2 Timeline and Arrangement migration APIs from the
  tarball, using [`../../fixtures/migration/v1-to-v2.json`](../../fixtures/migration/v1-to-v2.json).
- public TypeScript types for core migration APIs, React hooks, and Player React.
- React SSR import/render for `@vixeq/player-react`.
- Vite import of `@vixeq/player-react/styles.css`.
- packed-tarball builds for:
  - `apps/playground`
  - `examples/react-player`
  - `examples/vanilla-core`
  - `examples/arrangement-demo`
  - `examples/cycling-workout`
  - `examples/website-pulse`
  - `examples/website-svg`

## Publish Steps

1. Confirm npm authentication and the intended dist tag:
   `npm whoami` and `npm dist-tag ls @vixeq/core`.
2. Re-run `pnpm smoke:pack` after the final version bump.
3. Publish the three public packages with the beta tag from their package
   directories:
   `pnpm publish --tag beta --access public`.
4. Install the published beta versions in a clean temporary consumer and repeat
   the same smoke coverage.
5. Verify npm metadata for all packages:
   `npm view @vixeq/core@0.8.0-beta.1 version dist-tags exports`.
6. Record the published versions, npm URLs, migration fixture coverage, and any
   known beta caveats in the release notes.

## Deferred Release Gates

- CI API diff and API Extractor reports are R0.
- Coverage configuration and behavior-matrix gates are R1.
- Node/React/TypeScript/package-manager compatibility matrices are R2.
- Browser media/product E2E is R3.
