# 0.9 Release Checklist

This checklist publishes `0.9.0` as the final release-readiness line before
`1.0.0-rc.1`. It is a stable publish to the `latest` dist-tag, not a beta
prerelease.

## Preconditions

- R0-R6 are `done` in
  [`../plans/v1-collaboration-spec.md`](../plans/v1-collaboration-spec.md).
- R5 has an approved review in
  [`../reviews/r5-support-release-docs-claude.md`](../reviews/r5-support-release-docs-claude.md).
- Public package versions are bumped in lockstep to `0.9.0`:
  `package.json`, `packages/core/package.json`, `packages/react/package.json`,
  and `packages/player-react/package.json`.
- `CHANGELOG.md` has a `0.9.0 - YYYY-MM-DD` section covering the R0-R6 release
  readiness work.
- `CHANGELOG.md`, package READMEs, generated API docs, migration guides,
  [`../../SUPPORT.md`](../../SUPPORT.md), and this checklist describe the
  public behavior being released.
- API Extractor reports are current for all public packages.
- The working tree contains no unrelated release artifacts such as `.tgz`,
  `.tmp/`, copied example output, or generated probe files.
- The release commit is pushed to `main`, and the current `main` CI and Pages
  workflows are green.

## Required Local Gates

Run these from the repository root before tagging `v0.9.0`:

```sh
pnpm typecheck
pnpm build
pnpm test
pnpm api:check
pnpm behavior:check
pnpm smoke:pack
SMOKE_REACT_VERSION=18 pnpm smoke:pack
SMOKE_TS_VERSION=5.5 pnpm smoke:pack
pnpm test:e2e
node --test scripts/build-pages.test.mjs
```

Run `pnpm test:coverage` and record the result. For `0.9.0`, the known Core
coverage shortfall is non-blocking because R1 delivered the coverage
configuration and explicitly deferred CI wiring until the Core branch-coverage
gap is closed. This same gap must be resolved or explicitly re-decided before
`1.0.0-rc.1`.

## Publish Steps

1. Confirm npm registry state for all three packages:
   `npm view @vixeq/core dist-tags versions --json`,
   `npm view @vixeq/react dist-tags versions --json`, and
   `npm view @vixeq/player-react dist-tags versions --json`.
2. Confirm the version bump is lockstep:
   `rg '"version": "0.9.0"' package.json packages/*/package.json`.
3. Push the release commit to `main`.
4. Confirm CI and Pages workflows are green for that pushed commit.
5. Create and push the release tag:
   `git tag v0.9.0` and `git push origin v0.9.0`.
6. Confirm the GitHub Actions `Publish` workflow succeeds for `v0.9.0`.
7. Confirm npm metadata for all three packages. `latest` must point to `0.9.0`;
   the existing `beta` dist-tag should remain on `0.8.0-beta.1` unless a
   maintainer intentionally changes it.

## Post-Publish Registry Smoke

After the Publish workflow succeeds, install the published packages in a clean
temporary consumer and repeat the registry smoke coverage:

- `@vixeq/core@0.9.0` ESM import.
- `@vixeq/core@0.9.0` CJS require.
- public Timeline and Arrangement migration APIs from `@vixeq/core@0.9.0`.
- public TypeScript types for Core, React hooks, and Player React.
- React SSR import/render for `@vixeq/react@0.9.0` and
  `@vixeq/player-react@0.9.0`.
- Vite import of `@vixeq/player-react/styles.css`.
- package metadata and exports for all three packages.

Record the npm package URLs, dist-tags, local gate results, registry smoke
result, and the known non-blocking coverage gap in `CHANGELOG.md`.

## Deferred to 1.0 RC

- Core branch coverage still falls short of the configured R1 thresholds.
- `1.0.0-rc.1` must not publish until all 0.7-0.9 gates are green or a new
  approved decision updates the release contract.
