# 0.7 Beta Release Checklist

P8 prepares the packed-package smoke gate and release checklist. It does not publish
`0.7.0-beta.1`; publishing happens only after this checklist passes.

## Preconditions

- P0-P7F are `done` in `docs/plans/v1-collaboration-spec.md`.
- All public package versions are bumped in lockstep to the chosen prerelease
  version, for example `0.7.0-beta.1`.
- `CHANGELOG.md`, package READMEs, API docs, and migration docs describe the
  beta behavior being published.
- The working tree contains no unrelated release artifacts such as `.tgz`,
  `.tmp/`, or copied example output.

## Required Local Gates

Run these from the repository root before publishing a beta:

```sh
pnpm -r --no-bail typecheck
pnpm -r --no-bail test -- run
pnpm -r --no-bail build
pnpm smoke:pack
```

`pnpm smoke:pack` builds `@vixeq/core`, `@vixeq/react`, and
`@vixeq/player-react`, packs them with `pnpm pack`, installs those tarballs into
a temporary workspace under `.tmp/pack-smoke`, and verifies:

- `@vixeq/core` ESM import from the tarball.
- `@vixeq/core` CJS require from the tarball.
- public TypeScript types for core, React hooks, and Player React.
- React SSR import/render for `@vixeq/player-react`.
- Vite import of `@vixeq/player-react/styles.css`.
- packed-tarball builds for:
  - `apps/playground`
  - `examples/react-player`
  - `examples/vanilla-core`
  - `examples/arrangement-demo`
  - `examples/cycling-workout`
  - `examples/website-pulse`
  - `examples/website-svg`

## Publish Steps

1. Confirm npm authentication and the intended dist tag:
   `npm whoami` and `npm dist-tag ls @vixeq/core`.
2. Re-run `pnpm smoke:pack` after the final version bump.
3. Publish the three public packages with the beta tag from their package
   directories:
   `pnpm publish --tag beta --access public`.
4. Install the published beta versions in a clean temporary consumer and repeat
   the same smoke coverage.
5. Verify npm metadata for all packages:
   `npm view @vixeq/core@0.7.0-beta.1 version dist-tags exports`.
6. Record the published versions, npm URLs, and any known beta caveats in the
   release notes.

## Deferred Release Gates

- `publint` and Are The Types Wrong automation are release-gate follow-ups for
  R0/R2, not part of P8 implementation.
- npm/yarn/bun package-manager compatibility matrix is deferred to 0.9 R2.
- React 18/19 and TypeScript min/max version matrices are deferred to 0.9 R2.
- Browser E2E against the copied examples is outside P8; P8 requires build smoke
  only.
